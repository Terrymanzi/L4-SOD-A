## Project Overview

You will work in teams to design and build a **real-time chat application** using **Node.js, Express.js, and Socket.io**.

The application should allow multiple users to connect and communicate instantly through a web interface, demonstrating your understanding of backend development, real-time communication, and full-stack integration.

Your goal is to **build a functional, user-friendly chat system** that supports live messaging and showcases your technical skills.

---

## Objectives

- Apply **Node.js & Express.js** to build a backend server
- Implement **real-time communication using WebSockets (Socket.io)**
- Build a simple frontend interface for interaction (HTML, CSS & Optional JavaScript)
- Collaborate effectively using GitHub
- Present a working real-time system through a live demo

---

## Technical Requirements

### Required

- Use **Node.js + Express.js** for the server
- Use **Socket.io** for real-time messaging
- Implement:
  - User connection & disconnection handling
  - Sending & receiving messages in real time

- Build a basic frontend (HTML, CSS, optional JavaScript)
- Maintain **clean, modular code structure**
- Push all work to a **shared GitHub repository**
- Run the app locally or deploy (optional but encouraged)

### Bonus Points (Optional)

- Usernames or authentication
- Chat rooms or private messaging
- Message timestamps
- Typing indicators
- UI improvements (modern chat design)

---

## Team Structure

- Work in assigned groups
- Each member must contribute (visible via GitHub commits)
- Divide responsibilities (backend, frontend, testing, etc.)

---

## Timeline (3 Days)

### Day 1 - Setup & Core Functionality (4TH MAY)

- Set up Node.js + Express server
- Integrate Socket.io
- Implement basic message sending & receiving

### Day 2 - Features & UI (5TH MAY)

- Improve frontend UI
- Add features (usernames, rooms, timestamps, etc.)
- Handle edge cases (disconnects, empty messages)

### Day 3 - Testing & Presentation (6TH MAY)

- Test real-time behavior thoroughly
- Fix bugs and polish UI
- Prepare demo and GitHub repository

  - System flow (frontend → backend → socket events)
